# documentation.md

# codepot

TypeScript-first OpenAPI contract engine for building clean OpenAPI 3.1 specs and SDK-friendly metadata from code.

`codepot` lets you define your API contract using typed builders instead of manually writing YAML. It supports version contracts, resources, reusable properties, schemas, parameters, request bodies, responses, routes, default responses, shared primitives, enum detection, nullable/optional refs, and entity inheritance.

# 1. Overview

The package is designed around this principle:

```txt
Schemas define data types.
Parameters, request bodies, and responses wrap schemas.
Refs point to primary schema definitions.
SDK metadata is generated by the engine.
```

Instead of writing OpenAPI manually, you define contracts like this:

```ts
const users = v1.defineResource({
  key: 'users',
  name: 'User',
  basePath: '/users',
});

users.defineRoutes({
  listUsers: {
    method: HttpMethod.get,
    path: '/',
    summary: 'List users',
    responses: {
      200: userResponses.ref.UsersListOkResponse,
    },
  },
});
```

The package generates:

```yaml
paths:
  /users:
    get:
      operationId: listUsers
      x-codegen-group: users
      x-codegen-operation: list_users
      responses:
        '200':
          $ref: '#/components/responses/UsersListOkResponse'
```

# 6. Version contracts

A version contract represents one OpenAPI document.

```ts
const v1 = defineVersionContract({
  info: {
    title: 'RideRescue API',
    version: 'v1',
    description: 'RideRescue backend API',
    license: {
      name: 'MIT',
      identifier: 'MIT',
    },
  },
});
```

The `info` block maps to OpenAPI `info`.

```yaml
info:
  title: RideRescue API
  version: v1
  description: RideRescue backend API
  license:
    name: MIT
    identifier: MIT
```

A version can define:

```txt
resources
schema components
parameter components
request body components
response components
default responses
```

# 7. Resources

A resource groups endpoints and related schemas.

```ts
const users = v1.defineResource({
  key: 'users',
  name: 'User',
  basePath: '/users',
});
```

Fields:

```txt
key       internal/group key, used for x-codegen-group
name      human-readable resource name
basePath  base API path
```

A resource exposes:

```ts
users.defineProperties();
users.components.defineSchemas();
users.components.defineParameters();
users.components.defineRequestBodies();
users.components.defineResponses();
users.defineRoutes();
```

# 8. Properties

Properties are reusable field definitions.

Create a property registry:

```ts
const userProps = users.defineProperties();
```

Property registries can define:

```txt
.shared()
.forRef()
.entity()
```

# 9. Shared properties

Use `.shared()` for reusable fields that may be used across resources.

```ts
const sharedPrimitives = sharedProps.shared('SharedPrimitives', {
  mongoId: schema.primitive(z.string().regex(/^[a-f\d]{24}$/i), {
    description: 'Mongo ObjectId',
  }),
  dateTime: schema.primitive(z.string().datetime(), {
    description: 'ISO datetime',
  }),
  displayName: schema.primitive(z.string().min(1), {
    description: 'Display name',
  }),
});
```

Access:

```ts
sharedPrimitives.ref.mongoId;
sharedPrimitives.ref.dateTime;
sharedPrimitives.ref.displayName;
```

Shared fields automatically get shared SDK metadata:

```yaml
x-codegen-shared: true
x-codegen-group: shared
x-codegen-placement: global-shared
```

Primitive shared fields also get:

```yaml
x-codegen-kind: primitive
x-codegen-skip: true
```

# 10. `forRef` properties

Use `.forRef()` for reusable refs that are not full entities.

Example:

```ts
const userRefs = userProps.forRef('UserRefs', {
  userId: sharedPrimitives.ref.mongoId,
  status: schema.primitive(z.enum(['active', 'suspended', 'deleted']), {
    required: false,
  }),
});
```

Access:

```ts
userRefs.ref.userId;
userRefs.ref.status;
```

Typical use cases:

```txt
path IDs
query enum filters
query booleans
small reusable refs
```

# 11. Entities

Use `.entity()` for model-like objects.

```ts
const userEntity = userProps.entity<User>('User', {
  email: schema.primitive(z.string().email()),
  name: sharedPrimitives.ref.displayName,
  passwordHash: schema.primitive(z.string(), {
    access: SchemaAccess.secret,
  }),
});
```

Entity refs expose:

```ts
userEntity.ref.fields.email;
userEntity.ref.fields.name;
userEntity.ref.model;
userEntity.ref.publicModel;
userEntity.ref.selectedModel;
userEntity.ref.partialModel;
userEntity.ref.query.exact;
userEntity.ref.query.search;
userEntity.ref.query.exactSearch;
userEntity.ref.query.range;
userEntity.ref.query.in;
userEntity.ref.query.exists;
userEntity.ref.query.sort;
```

Entity models:

```txt
model          full model
publicModel    excludes secret/internal/system fields by default
selectedModel  respects select rules
partialModel   all fields optional
```

# 12. Abstract entities and inheritance

Use abstract entities for common model fields like:

```txt
id
createdAt
updatedAt
tenantId
deletedAt
```

## Define abstract base entity

```ts
type BaseEntity = {
  id: string;
  createdAt: string;
  updatedAt: string;
};

const baseEntity = sharedProps.entity<BaseEntity>(
  'BaseEntity',
  {
    id: sharedPrimitives.ref.mongoId,
    createdAt: sharedPrimitives.ref.dateTime,
    updatedAt: sharedPrimitives.ref.dateTime,
  },
  {
    abstract: true,
  },
);
```

## Extend exact entity ref

The locked inheritance rule is:

```ts
extends: baseEntity.ref
```

Do not pass the full registry:

```ts
extends: baseEntity
```

Correct:

```ts
const userEntity = userProps.entity<User>(
  'User',
  {
    email: schema.primitive(z.string().email()),
    name: sharedPrimitives.ref.displayName,
  },
  {
    extends: baseEntity.ref,
  },
);
```

The inheriting entity can access inherited fields:

```ts
userEntity.ref.fields.id;
userEntity.ref.fields.createdAt;
userEntity.ref.fields.updatedAt;
userEntity.ref.fields.email;
```

Expected OpenAPI metadata:

```yaml
UserPublicModel:
  x-codegen-inherits:
    - ref: '#/components/schemas/BaseEntity'
      fields:
        - id
        - createdAt
        - updatedAt
```

The inherited model remains flat:

```yaml
UserPublicModel:
  type: object
  properties:
    id:
      $ref: '#/components/schemas/SharedPrimitivesMongoId'
    createdAt:
      $ref: '#/components/schemas/SharedPrimitivesDateTime'
    updatedAt:
      $ref: '#/components/schemas/SharedPrimitivesDateTime'
    email:
      $ref: '#/components/schemas/UserEmail'
```

# 13. Access, select, required, nullable

## Required

By default, fields are required.

```ts
email: schema.primitive(z.string().email());
```

To make a field optional:

```ts
phone: schema.primitive(z.string(), {
  required: false,
});
```

## Nullable

To allow `null`:

```ts
phone: schema.primitive(z.string(), {
  required: false,
  nullable: true,
});
```

Expected OpenAPI:

```yaml
phone:
  anyOf:
    - type: string
    - type: 'null'
```

## Access

```ts
passwordHash: schema.primitive(z.string(), {
  access: SchemaAccess.secret,
});
```

Supported access levels:

```ts
SchemaAccess.public;
SchemaAccess.auth;
SchemaAccess.private;
SchemaAccess.secret;
SchemaAccess.internal;
SchemaAccess.system;
```

Recommended meaning:

```txt
public    visible in public models
auth      visible in authenticated models
private   hidden by default
secret    hidden by default
internal  hidden by default
system    hidden by default unless selected
```

## Select

Override default select behavior:

```ts
id: schema.primitive(z.string(), {
  access: SchemaAccess.internal,
  select: true,
});
```

or:

```ts
passwordHash: schema.primitive(z.string(), {
  access: SchemaAccess.secret,
  select: false,
});
```

# 14. Query behavior

Query behavior lets entities expose reusable query refs.

```ts
email: schema.primitive(z.string().email(), {
  query: {
    methods: [QueryBehavior.exactSearch, QueryBehavior.search],
    sort: true,
  },
});
```

Supported query methods:

```ts
QueryBehavior.exact;
QueryBehavior.search;
QueryBehavior.exactSearch;
QueryBehavior.range;
QueryBehavior.in;
QueryBehavior.exists;
```

Entity query refs:

```ts
userEntity.ref.query.exact;
userEntity.ref.query.search;
userEntity.ref.query.exactSearch;
userEntity.ref.query.range;
userEntity.ref.query.in;
userEntity.ref.query.exists;
userEntity.ref.query.sort;
```

Deep field query refs are intentionally not supported.

Do not use:

```ts
userEntity.ref.sort.email;
```

Use:

```ts
userEntity.ref.query.sort;
```

# 15. Component buckets

The package separates OpenAPI components into proper buckets.

```txt
components.schemas
components.parameters
components.requestBodies
components.responses
```

Actual data types live in:

```txt
components.schemas
```

OpenAPI wrappers live in:

```txt
components.parameters
components.requestBodies
components.responses
```

This keeps SDK generation clean.

# 16. Schema components

Define reusable DTO schemas.

```ts
const sharedSchemas = v1.components.defineSchemas({
  ApiMessage: {
    success: sharedPrimitives.ref.success,
    message: sharedPrimitives.ref.message,
  },
  ValidationError: {
    success: sharedPrimitives.ref.success,
    message: sharedPrimitives.ref.message,
  },
});
```

Resource-level schema components:

```ts
const userSchemas = users.components.defineSchemas({
  CreateUserBody: {
    email: userEntity.ref.fields.email,
    name: userEntity.ref.fields.name,
  },
  UpdateUserBody: {
    email: userEntity.ref.fields.email.optional(),
    name: userEntity.ref.fields.name.optional(),
  },
});
```

Schema components emit:

```yaml
x-codegen-kind: dto
```

Version-level schema components are shared:

```yaml
x-codegen-shared: true
x-codegen-group: shared
x-codegen-placement: global-shared
```

Define reusable OpenAPI parameters.

```ts
const sharedParams = v1.components.defineParameters({

    name: 'page',

  },
  LimitQueryParam: {
    name: 'limit',
    in: ParameterLocation.query,
    schema: sharedPrimitives.ref.limit,
  },
});
```

Parameter locations:

```ts
ParameterLocation.path;
ParameterLocation.query;
ParameterLocation.header;
ParameterLocation.cookie;
```

Parameter components emit:

```yaml
x-codegen-kind: dto
x-codegen-skip: true
```

They are wrappers, not SDK model classes.

# 20. Default responses

Define version-level default responses once:

```ts
v1.setDefaultResponses({
  400: sharedResponses.ref.BadRequestResponse,
  401: sharedResponses.ref.UnauthorizedResponse,
  403: sharedResponses.ref.ForbiddenResponse,
  422: sharedResponses.ref.ValidationErrorResponse,
  500: sharedResponses.ref.InternalServerErrorResponse,
});
```

Every route receives these defaults unless route responses override them.

Example route:

```ts
responses: {
  200: userResponses.ref.UserOkResponse,
  404: sharedResponses.ref.NotFoundResponse,
}
```

Final output:

```yaml
responses:
  '200':
    $ref: '#/components/responses/UserOkResponse'
  '400':
    $ref: '#/components/responses/BadRequestResponse'
  '401':
    $ref: '#/components/responses/UnauthorizedResponse'
  '403':
    $ref: '#/components/responses/ForbiddenResponse'
  '404':
    $ref: '#/components/responses/NotFoundResponse'
  '422':
    $ref: '#/components/responses/ValidationErrorResponse'
  '500':
    $ref: '#/components/responses/InternalServerErrorResponse'
```

# 21. Routes

Routes are defined per resource.

Simple shape:

```ts
users.defineRoutes({
  listUsers: {
    method: HttpMethod.get,
    path: '/',
    summary: 'List users',
    responses: {
      200: userResponses.ref.UsersListOkResponse,
    },
  },
});
```

Nested shape with shared path parameters:

```ts
users.defineRoutes({
  parameters: {
    '/:userId': [userParams.ref.UserIdPathParam],
  },
  routes: {
    getUserById: {
      method: HttpMethod.get,
      path: '/:userId',
      query: [sharedParams.ref.FieldsQueryParam, sharedParams.ref.PopulateQueryParam],
      responses: {
        200: userResponses.ref.UserOkResponse,
        404: sharedResponses.ref.NotFoundResponse,
      },
    },
  },
});
```

Supported route fields:

```txt
method
path
summary
description
params
query
parameters
body
response
responses
operationId
tags
meta
```

# 22. Path parameters vs query parameters

Use this rule:

```txt
Path parameters may be shared at path-item level.
Query parameters belong to individual operations.
```

## Path-level shared params

```ts
users.defineRoutes({
  parameters: {
    '/:userId': [userParams.ref.UserIdPathParam],
  },
  routes: {
    getUserById: {
      method: HttpMethod.get,
      path: '/:userId',
      responses: {
        200: userResponses.ref.UserOkResponse,
      },
    },
  },
});
```

Output:

```yaml
/users/{userId}:
  parameters:
    - $ref: '#/components/parameters/UserIdPathParam'
```

## Operation-level query params

```ts
listUsers: {
  method: HttpMethod.get,
  path: '/',
  query: [
    sharedParams.ref.PageQueryParam,
    sharedParams.ref.LimitQueryParam,
    sharedParams.ref.SearchQueryParam,
  ],
}
```

Output:

```yaml
/users:
  get:
    parameters:
      - $ref: '#/components/parameters/PageQueryParam'
      - $ref: '#/components/parameters/LimitQueryParam'
      - $ref: '#/components/parameters/SearchQueryParam'
```

# 23. Ref usage methods

Every ref supports usage modifiers:

```ts
.optional()
.required()
.nullable()
.nonNullable()
```

They can be chained:

```ts
phone: userEntity.ref.fields.phone.optional().nullable();
```

Meaning:

```txt
phone is not required
phone may be null
```

Usage modifiers affect the **usage site**, not the original schema.

Example:

```ts
avatarUrl: sharedPrimitives.ref.url.optional().nullable();
```

Expected output:

```yaml
avatarUrl:
  anyOf:
    - $ref: '#/components/schemas/SharedPrimitivesUrl'
    - type: 'null'
```

And `avatarUrl` is excluded from `required`.

# 24. Ref-as-pointer behavior

Refs behave like pointers.

If a field uses another field ref, the compiler should not create a duplicate primitive schema.

Example:

```ts
id: sharedPrimitives.ref.mongoId;
```

The generated model property points directly to:

```yaml
id:
  $ref: '#/components/schemas/SharedPrimitivesMongoId'
```

Not:

```yaml
id:
  $ref: '#/components/schemas/UserId'
```

This avoids duplicate primitive schemas and unresolved refs.

If usage changes:

```ts
id: sharedPrimitives.ref.mongoId.nullable();
```

Output:

```yaml
id:
  anyOf:
    - $ref: '#/components/schemas/SharedPrimitivesMongoId'
    - type: 'null'
```

# 25. SDK metadata rules

The engine emits SDK metadata automatically.

## Primitive fields

```yaml
x-codegen-kind: primitive
x-codegen-skip: true
```

Primitive schemas are OpenAPI building blocks and should not become standalone SDK classes.

## Enum fields

If a primitive uses `z.enum(...)`, it emits:

```yaml
x-codegen-kind: enum
x-codegen-skip: false
```

Example:

```ts
status: schema.primitive(z.enum(['active', 'suspended', 'deleted']));
```

Expected:

```yaml
UserStatus:
  type: string
  enum:
    - active
    - suspended
    - deleted
  x-codegen-kind: enum
  x-codegen-skip: false
```

## Schema components

```yaml
x-codegen-kind: dto
```

## Parameter/request body/response wrappers

```yaml
x-codegen-kind: dto
x-codegen-skip: true
```

## Shared items

```yaml
x-codegen-shared: true
x-codegen-group: shared
x-codegen-placement: global-shared
```

## Operations

```yaml
operationId: listUsers
x-codegen-group: users
x-codegen-operation: list_users
```

# 26. Recommended full example

```ts
import { z } from 'zod';
import { definePackageConfig, defineVersionContract, schema, HttpMethod, SchemaAccess, QueryBehavior, ParameterLocation } from 'codepot';

const v1 = defineVersionContract({
  info: {
    title: 'RideRescue API',
    version: 'v1',
    description: 'RideRescue backend API',
    license: { name: 'MIT', identifier: 'MIT' },
  },
});

const shared = v1.defineResource({
  key: 'shared',
  name: 'Shared',
  basePath: '/_shared',
});

const sharedProps = shared.defineProperties();

const sharedPrimitives = sharedProps.shared('SharedPrimitives', {
  mongoId: schema.primitive(z.string().regex(/^[a-f\d]{24}$/i), {
    description: 'Mongo ObjectId',
  }),
  dateTime: schema.primitive(z.string().datetime(), {
    description: 'ISO datetime',
  }),
  displayName: schema.primitive(z.string().min(1), {
    description: 'Display name',
  }),
  success: schema.primitive(z.boolean()),
  message: schema.primitive(z.string()),
  page: schema.primitive(z.number().int().positive(), { required: false }),
  limit: schema.primitive(z.number().int().positive(), { required: false }),
  sort: schema.primitive(z.string(), { required: false }),
  fields: schema.primitive(z.string(), { required: false }),
  populate: schema.primitive(z.string(), { required: false }),
  search: schema.primitive(z.string(), { required: false }),
});

type BaseEntity = {
  id: string;
  createdAt: string;
  updatedAt: string;
};

const baseEntity = sharedProps.entity<BaseEntity>(
  'BaseEntity',
  {
    id: sharedPrimitives.ref.mongoId,
    createdAt: sharedPrimitives.ref.dateTime,
    updatedAt: sharedPrimitives.ref.dateTime,
  },
  {
    abstract: true,
  },
);

const sharedSchemas = v1.components.defineSchemas({
  ApiMessage: {
    success: sharedPrimitives.ref.success,
    message: sharedPrimitives.ref.message,
  },
  ValidationError: {
    success: sharedPrimitives.ref.success,
    message: sharedPrimitives.ref.message,
  },
  PaginatedMeta: {
    page: sharedPrimitives.ref.page,
    limit: sharedPrimitives.ref.limit,
  },
});

const sharedParams = v1.components.defineParameters({
  PageQueryParam: {
    name: 'page',
    in: ParameterLocation.query,
    schema: sharedPrimitives.ref.page,
  },
  LimitQueryParam: {
    name: 'limit',
    in: ParameterLocation.query,
    schema: sharedPrimitives.ref.limit,
  },
  SortQueryParam: {
    name: 'sort',
    in: ParameterLocation.query,
    schema: sharedPrimitives.ref.sort,
  },
  FieldsQueryParam: {
    name: 'fields',
    in: ParameterLocation.query,
    schema: sharedPrimitives.ref.fields,
  },
  PopulateQueryParam: {
    name: 'populate',
    in: ParameterLocation.query,
    schema: sharedPrimitives.ref.populate,
  },
  SearchQueryParam: {
    name: 'search',
    in: ParameterLocation.query,
    schema: sharedPrimitives.ref.search,
  },
});

const sharedResponses = v1.components.defineResponses({
  BadRequestResponse: {
    description: 'Bad request',
    schema: sharedSchemas.ref.ApiMessage,
  },
  UnauthorizedResponse: {
    description: 'Unauthorized',
    schema: sharedSchemas.ref.ApiMessage,
  },
  ForbiddenResponse: {
    description: 'Forbidden',
    schema: sharedSchemas.ref.ApiMessage,
  },
  NotFoundResponse: {
    description: 'Not found',
    schema: sharedSchemas.ref.ApiMessage,
  },
  ConflictResponse: {
    description: 'Conflict',
    schema: sharedSchemas.ref.ApiMessage,
  },
  ValidationErrorResponse: {
    description: 'Validation failed',
    schema: sharedSchemas.ref.ValidationError,
  },
  InternalServerErrorResponse: {
    description: 'Internal server error',
    schema: sharedSchemas.ref.ApiMessage,
  },
});

v1.setDefaultResponses({
  400: sharedResponses.ref.BadRequestResponse,
  401: sharedResponses.ref.UnauthorizedResponse,
  403: sharedResponses.ref.ForbiddenResponse,
  422: sharedResponses.ref.ValidationErrorResponse,
  500: sharedResponses.ref.InternalServerErrorResponse,
});

const users = v1.defineResource({
```

# 29. Best practices

## Use shared primitives for global field types

Good:

```ts
id: sharedPrimitives.ref.mongoId;
createdAt: sharedPrimitives.ref.dateTime;
name: sharedPrimitives.ref.displayName;
```

Avoid redefining the same primitive in every resource.

## Use exact inheritance refs

Good:

```ts
extends: baseEntity.ref
```

Avoid:

```ts
extends: baseEntity
```

## Keep schemas as data shapes

Good:

```ts
users.components.defineSchemas({
  CreateUserBody: {
    email: userEntity.ref.fields.email,
  },
});
```

## Keep responses as wrappers

Good:

```ts
users.components.defineResponses({
  UserOkResponse: {
    description: 'User response',
    schema: userEntity.ref.publicModel,
  },
});
```

## Use path-level params only for path params

Good:

```ts
users.defineRoutes({
  parameters: {
    '/:userId': [userParams.ref.UserIdPathParam],
  },
  routes: {},
});
```

## Put query params on operations

Good:

```ts
listUsers: {
  query: [
    sharedParams.ref.PageQueryParam,
    sharedParams.ref.SearchQueryParam,
  ],
}
```

# 30. Common errors and fixes

## Error: unresolved ref like `UserId`

Cause:

```txt
A field points to another ref, but compiler emitted alias ref instead of primary target ref.
```

Fix:

```txt
Ensure ref-as-pointer behavior is enabled.
Model/component fields should use fieldRef.targetRefId ?? fieldRef.id.
Alias primitive schemas should not be emitted.
```

Expected:

```yaml
id:
  $ref: '#/components/schemas/SharedPrimitivesMongoId'
```

Not:

```yaml
id:
  $ref: '#/components/schemas/UserId'
```

## Error: fake fields like `kind`, `propertyKey`, `optional`

Cause:

```txt
Inheritance merged a PropertyRef object itself instead of EntityPropertyRefs.fields.
```

Fix:

```txt
Use extends: baseEntity.ref.
Do not use extends: baseEntity.
```

## Error: `NamedEntityPropertyRegistry` not assignable to `undefined`

Cause:

```txt
Entity generic signature locked extends to undefined.
```

Fix package signature so `TExtends` is inferred from the third argument.

Official usage remains:

```ts
extends: baseEntity.ref
```

## Warning: component is never used

Examples:

```txt
PaginatedMeta is never used
UserPasswordHash is never used
```

Fix by either using the component or removing it.

Example using `PaginatedMeta`:

```ts
UsersListOk: {
  success: sharedPrimitives.ref.success,
  message: sharedPrimitives.ref.message,
  meta: sharedSchemas.ref.PaginatedMeta,
}
```

Secret fields like `passwordHash` may be emitted but unused if not included in public models. The compiler may skip secret/internal/system primitives unless directly referenced.

# Final mental model

Use the package like this:

```txt
1. Define version.
2. Define shared primitives.
3. Define abstract base entities if needed.
4. Define shared schemas/params/responses.
5. Define resource entities.
6. Define resource schemas.
7. Define resource params.
8. Define request bodies.
9. Define responses.
10. Define routes.
11. Generate OpenAPI.
12. Lint OpenAPI.
13. Feed OpenAPI into SDK generators.
```

The engine’s job is to produce OpenAPI that is:

```txt
valid
deduplicated
ref-safe
SDK-friendly
metadata-rich
easy to trace
```

The user’s job is to define reusable refs clearly and route behavior explicitly.
