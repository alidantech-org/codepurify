# Global Context

Global context provides system-wide data and configuration.

## Global Structure

```hbs
{| global.templates.* |} // Template metadata {| global.config.* |} // System
configuration {| global.utils.* |} // Utility functions
```

## Template Metadata

```hbs
{| global.templates.current.name |} // "user-dto" {|
global.templates.current.type |} // "dto" {|
global.templates.current.output_path |} // "./output/user.dto.ts" {|
global.templates.all.count |} // Total templates
```

## System Configuration

```hbs
{| global.config.project.name |} // "MyProject" {|
global.config.output.directory |} // "./generated" {|
global.config.validation.enabled |} // true
```

## Utility Functions

```hbs
{| global.utils.string.to_pascal |} // Convert to PascalCase {|
global.utils.path.normalize |} // Normalize file paths
```

## When to Use Global Context

Use global context for:

- Template metadata and naming
- System-wide configuration
- Shared utilities across templates
- Cross-template references

## Example Usage

```hbs
// Template header comment /* * Generated by {| global.templates.current.name |}
* Project: {| global.config.project.name |} */ // Conditional validation {|#if
global.config.validation.enabled|} import { validate } from './validation';
{|/if|} // File path generation const outputPath = '{|
global.config.output.directory |}/{| entity.names.casing.kebab |}.ts';
```

## Global vs Entity Context

- **Entity context** - Data about the current entity being processed
- **Global context** - System-wide data and configuration

```hbs
// Entity context - specific to this entity {| entity.names.casing.pascal |} //
Global context - system-wide {| global.config.project.name |} {|
global.templates.current.name |}
```

Global context provides the foundation for consistent, system-aware template generation.
